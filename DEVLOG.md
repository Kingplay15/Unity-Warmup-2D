**Devlog – Phase 1 Warmup**

**Context**

After a long break from Unity, I completed a small 2D tutorial project as a warm-up exercise.

The goal was not creativity or polish, but to:

- Rebuild familiarity with Unity interface

- Revisit core 2D workflow

- Restore development habit

**What I Did**

- Implemented basic player movement

- Implemented simple collision and interaction

- Practiced prefab usage

- Revisited basic scripting structure

**Observations**

Most of the process was straightforward because I followed a tutorial.
I did not need to design or solve complex problems.

_This confirms:_

- I still understand core Unity basics

- I can quickly regain technical rhythm

_However:_

- I did not exercise creativity

- I did not challenge system design thinking

**Lessons Learned**

Warm-up projects should focus on rebuilding momentum, not proving skill.

The next phase must include:

- Independent problem solving

- System design thinking

- Basic combat prototype without full tutorial guidance

**Next Phase**

Phase 2 will focus on building a simple combat prototype from scratch (without fully following tutorial instructions).
